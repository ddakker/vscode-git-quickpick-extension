## (선택, 낮은 우선순위) 변경/스태시 섹션도 webview로 통합
- **What:** 현재 하이브리드에서 네이티브 트리로 남기는 변경 파일·스태시 섹션을
  나중에 webview로 옮겨 패널 전체를 단일 webview로 통일.
- **Why:** 레이아웃 완전 자유도(한 면 자유 배치). 단 **사용자 판단상 현재는 불필요**
  — 변경/스태시는 커밋-필드 색상/고정폭 요구가 없고, 파일 체크박스·멀티셀렉트·접근성은
  네이티브 트리가 더 낫다.
- **Pros:** UI 통일감, 레이아웃 자유.
- **Cons:** 네이티브 체크박스/멀티셀렉트/접근성/파일아이콘을 직접 재구현(조용한 오동작 위험),
  fakeItem 속성 계약을 파일/스태시 명령까지 확장.
- **Context:** 2026-06-19 plan-eng-review. 1차 범위 = 히스토리 + 브랜치-히스토리만 webview
  (커밋 표시가 나타나는 곳). 변경/스태시 트리 유지는 의도된 선택이며 회피가 아니다.
- **Depends on / blocked by:** 히스토리/브랜치 webview 안정화. 실제 필요성이 확인될 때만 착수.
