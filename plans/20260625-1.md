# 작업 내용 (2026-06-25 세션 1)

## 커밋 펼침 상태가 전체 섹션에 동시 적용되는 버그 수정

**증상:** 히스토리 / 로컬 브랜치 / 리모트 브랜치 중 어느 섹션에서 커밋을 펼쳐도
세 섹션 모두 파일 목록이 동시에 펼쳐짐

**원인:** `_expandedCommits`(Set)이 커밋 해시만 저장  
→ 동일 해시가 여러 섹션에 존재할 때 `commitFiles[hash]`가 채워지면  
`renderCommitTable`이 섹션 구분 없이 모든 곳에서 `expanded=true`로 렌더링

**수정:** 펼침 상태를 `"섹션|해시"` 복합 키로 관리

### 변경 파일

| 파일 | 변경 내용 |
|---|---|
| `src/webview/provider.js` | `toggleCommit` 키를 `msg.section + '|' + msg.hash`로 변경 |
| `src/webview/build-state.js` | `expandedCommitKeys`를 state에 추가, 해시 추출 시 `|` 뒤 부분 사용 |
| `lib/webview-html.js` | `renderCommitTable`에 `expandedKeys`/`section` 파라미터 추가, 섹션별 해시 집합으로 필터링. 클릭/키보드 핸들러에서 `section` 전송 |

### 하위 호환

`expandedCommitKeys`가 비어 있으면 기존 동작(`commitFiles` 전체 사용)으로 폴백 → 기존 테스트 유지
